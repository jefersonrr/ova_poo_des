SUBTEMA 1.3 Conceptos básicos de Objetos

Características -> Atributos

Las propiedades o atributos son elementos que forman parte de una clase en programación orientada a objetos. Cada propiedad representa una característica o una cualidad de un objeto de la clase. Por ejemplo, si estamos creando una clase llamada "perro", una propiedad podría ser el nombre del perro, otro podría ser su peso, y otro podría ser su altura. Cada objeto de la clase "perro" tendrá su propio conjunto de valores para cada una de estas propiedades.

Métodos

Los métodos en programación orientada a objetos son funciones que se asocian a una clase y que pueden ser usadas por las instancias de esa clase. Los métodos se usan para definir la funcionalidad de una clase y para proporcionar una forma de acceder a los datos de una instancia de esa clase.

Estado

El estado en programación orientada a objetos se refiere a la información que se almacena en una instancia de una clase. Cuando se crea una instancia de una clase, se asigna espacio en memoria para almacenar los datos de esa instancia. Esta información es lo que se conoce como el estado de la instancia.

Identidad

La identidad en programación orientada a objetos se refiere a la forma en que se identifica una instancia de una clase. Cada instancia de una clase tiene una identidad única que la distingue de las demás instancias de la misma clase. La identidad de una instancia se puede determinar mediante una dirección de memoria o un identificador único. La identidad de una instancia es importante porque permite a un programa diferenciar entre distintas instancias de la misma clase y tratarlas de forma individual.

Tipos de objetos en el mundo real

En programación orientada a objetos se pueden llegar a describir una gran cantidad de objetos del mundo real, definiendo sus propiedades y sus métodos. Asimismo se debe hacer la distinción entre objetos físicos del mundo real y objetos abstractos que existen conceptualmente pero no físicamente como por ejemplo una idea.
